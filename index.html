Variables

The variables that you see in JavaScript are similar to the ones you see in math. They are used to substitute values without explicitly using said values. The names of variables are called "identifiers", and must follow certain rules.

The name/identifier for a variable in JavaScript must start with either a letter, an underscore "_", or a dollar sign "$". The characters after the first one can contain the digits 0 to 9 and also include uppercase and lowercase characters from "a" to "Z". You can also use ISO 8859-1 or Unicode letters such as å and ü in naming.

Here are some examples of legal identifiers: temp, color2, _weather, $variable2, and hello_world.

Declaring Variables

In order to "create" a variable you must first declare it.

There are four common ways to declare variables:

You can use the keyword "var". For example:

var x = 2;

This syntax of "var" can be used to declare both local and global variables.

By assigning a value to a identifier without any keywords, it can create a variable that is always global. For example:

x = 2;

Depending on the setup, this could generate a JavaScript error. This way of declaring variables should be avoided.

The keyword "let" is commonly used to declare variables whose values are subject to change. For example:

let x = 2;

x = 3;

The first line of code declares a new variable called "x" and assigns it the value of 2. This means that every reference to "x" will automatically substitute "x" with 2. The second line of code reassigns the variable "x" a new value of 3. Now every reference to "x" will result in a 3.

The last keyword I will be covering will be the keyword "const". For example:

const x = 2;

The "const" keyword lets the computer know that the value assigned to "x" must not be reassigned. The following statement after the "const x = 2;" declaration would result in a JavaScript warning:

x = 3;
Scope

Scope is the name for where a variable lives. There are three types of scopes in JavaScript:

  • Block
  • Function
  • Global

Variables declared in a {} block live only within the brackets. That means that you can only access those within the block or else an error occurs. For example:

{
let x = 4 // this is valid and x can be accessed here
}
// console.log(x) would not work and would cause an error

Variables that are declared within a function are in the local scope. It cannot be accessed outside of the function. For example:

function exampleFunction() {
let variableName = "one" // this can be accessed inside the function
}
// console.log(variableName) would not work here and would throw an error

Here are two reasons why local variables are used:

  • they make memory management more efficient. Once the function ends, the space for the local variables is reallocated.
  • they reduce problems down the line when you want to name a variable, but the variables name is already taken. You might also forget that a variable is storing some other value that do you not need.

A variable declared outside of a function or a block is called global. This means that is can be accessed and referenced anywhere in the file. For example:

let variableName = "one"
// variableName can be accessed here
function exampleFunction() {
// variableName can also be accessed here
}
